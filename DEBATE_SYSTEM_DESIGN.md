## 二、v5 七阶段流水线

```
Top50（V3 基本面排序 + 强制纳入 001309/600522）
  │
  ▼ 【阶段1】增量信息采集 (incremental.py)
  │   · 实时财务摘要 (akshare get_fundamentals)
  │   · 近30天新闻 (WebSearch缓存 → 东方财富API回退, 按公司名称搜索)
  │   · 竞争与增长 (护城河/增长驱动/逆风, 完整输出不截断)
  │   · 动态信号 (5日/20日涨幅/动量加速/量能异动/新高突破/资金流趋势/波动率)
  │   · K线10日走势 + 资金流5日明细
  │   · LLM事件摘要 (带偏多/偏空标签)
  │
  ▼ 【阶段2】三分析师报告 (analysts.py)
  │   · 技术面分析师 — 侧重动态信号+K线走势
  │   · 资金面分析师 — 侧重资金流明细+主力趋势
  │   · 基本面分析师 — 侧重实时财务+新闻+竞争增长 (deep=True)
  │
  ▼ 【阶段3】海选 (screen_round1) — hybrid 模式
  │   · V3 Top-6 直接保送进决赛 (守住龙头基本盘)
  │   · 剩余44只经 LLM 海选竞争 4 个名额
  │   · 海选 prompt 注入 V3 先验 + 增量信息
  │
  ▼ 【阶段4-5】三轮 claim 驱动辩论 (debaters.py)
  │   · 第1轮: 基于增量信息建 claim (全量信息灌入)
  │   · 第2轮: 针对证据本身反驳 (精简信息, 聚焦攻防)
  │   · 第3轮: 结合时间窗口+资金趋势+位置定排序
  │   · 多头: 强制证据引用(日期/数值), 催化新鲜度维度
  │   · 空头: 5种精准打击(催化过时/资金背离/量价背离/高位透支/增速证伪)
  │
  ▼ 【阶段6】终极PK (judges.py)
  │   · 条件性排名调整: 仅当有硬证据时才调整 (max_rise=3, max_drop=3)
  │   · 对称调整: 多头高置信度 claim 可上调, 空头硬风险标签可下调
  │   · 量价背离/高位透支等硬风险标签 → 直接降权
  │
  ▼ 【阶段7】报告输出 (reporter.py)
      · 最终 TOP10 排名 + 每只股票的 claim 账本 + 调整理由
```

## 三、增量信息架构（核心创新）

### 信息增量问题诊断

v4 版本辩论对排名影响有限（Top5 几乎不变），根因：
1. **V3 静态数据 + 技术/资金信号对 30 天预测力不足**
2. **在最核心的基本面数据上没有新信息进来**（fundamentals JSON 数据滞后）
3. **辩论阶段看不到分析师报告和实时数据**

### 增量信息采集层（incremental.py）

每只候选股的 brief 包含 5 个板块：

| 板块 | 数据来源 | 内容 | 对30天预测的贡献 |
|------|---------|------|----------------|
| **最新财务摘要** | akshare `get_fundamentals` | 实时财报数据（比 fundamentals JSON 更新） | 业绩增速验证 |
| **近期新闻** | WebSearch缓存 → 东方财富API | 按公司名称搜索，按时间线降序，过滤泛资讯 | 催化时效判断 |
| **竞争与增长** | fundamentals JSON | 护城河/增长评分/驱动/逆风/业务概览/行业地位/财务核心指标 | 增长持续性 |
| **动态信号** | K线+资金流计算 | 5日/20日涨幅、动量加速、量能异动、新高突破、资金流趋势、波动率 | 短期动能 |
| **近期事件** | LLM 批量生成 | 带偏多/偏空标签的事件摘要 | 事件驱动 |

### 新增数据维度（v5 vs v4）

| 维度 | v4 | v5 |
|------|----|----|
| K线走势 | 仅标签(加速/放量) | **10日走势(每日收盘价+涨跌幅) + 关键价位** |
| 资金流 | 仅趋势标签 | **5日每日主力净流入明细** |
| 财务数据 | fundamentals JSON(滞后) | **akshare 实时财务 + 财务亮点(含同比增速)** |
| 新闻 | 无 | **按公司名称搜索 + WebSearch缓存 + 时间线排序** |
| 增长驱动 | 截断60字 | **完整输出(驱动1/2/3 + 逆风)** |
| 业务概览 | 无 | **营收结构/产品占比/毛利率变动** |
| 行业地位 | 无 | **市占率/客户结构/竞争格局** |

### 回测模式处理

回测时禁用实时数据避免未来函数：
- 跳过【最新财务摘要】和【近期新闻】
- 仅保留【竞争与增长】+【动态信号】+【近期事件】

## 四、海选模式（screen_mode）

### 三种模式对比（2窗口回测验证）

| 模式 | T5平均收益 | T10平均收益 | 黑马优势 | 黑马胜率 |
|------|-----------|-----------|---------|---------|
| **promote** (V3 Top-K保送) | +30.88% | +18.60% | — | — |
| **llm** (全海选) | +15.07% | +15.16% | -5.74% | 0% |
| **hybrid** (保送6+海选4) | +30.42% | **+22.14%** | **+11.82%** | **100%** |

### 决策：采用 hybrid 模式

- **V3 Top-6 保送**守住龙头基本盘，避免 llm 模式的灾难性误杀
- **4 个海选名额**让增量信息驱动的黑马进入决赛（如 000988、600522、001389）
- T5 收益几乎无损（-0.46%），T10 收益提升 3.54%
- 黑马 100% 胜率，换入股平均比换出股强 11.82%

### 配置

```python
PickerGraph(
    screen_mode="hybrid",   # promote | llm | hybrid
    debate_top_k=10,        # 决赛名额
    force_k=6,              # hybrid模式下V3保送数量
)
```

## 五、辩论策略优化

### 三轮目标设计

| 轮次 | 目标 | 信息量 | 核心任务 |
|------|------|--------|---------|
| 第1轮 | 基于增量信息建 claim | 全量 | 每只股票建立 2-3 个核心 claim |
| 第2轮 | 针对证据本身反驳 | 精简 | 攻击对方 claim 的证据质量/时效 |
| 第3轮 | 结合时间窗口定排序 | 精简 | 催化时效+资金趋势+位置→排序 |

### 多头策略（BULL）

- **强制证据引用**：每个 claim 的 evidence 必须引用带日期/数值的具体数据点
- **催化新鲜度**：近 2 周有新催化 > 1-2 月前的旧催化
- **禁止空泛表述**："基本面好/龙头地位"等无数据支撑的论点无效

### 空头策略（BEAR）

- **5 种精准打击**：
  1. 催化过时：最近催化在 1 个月前，30 天窗口内无新催化
  2. 资金背离：股价涨但主力持续净流出
  3. 量价背离：价格新高但成交量萎缩
  4. 高位透支：短期涨幅已超 30%，估值远超行业均值
  5. 增速证伪：最新财报增速不及预期或环比下滑
- **禁止无效攻击**：在 AI 行情中，"大客户集中""估值高"是正向信号，不允许作为空头论据

### 排名调整机制

- **条件性调整**：仅当有硬证据时才调整，最多 ±3 位
- **对称调整**：多头高置信度 claim 可上调至 max_rise 位，空头硬风险标签可下调至 max_drop 位
- **硬风险标签**：量价背离、高位透支、催化过时 → 直接降权

## 六、新闻缓存机制

### 架构

```
WebSearch (手动预填充)
  │
  ▼ news_cache.json (按股票代码缓存)
  │
  ▼ incremental.py 优先读取缓存
  │   缓存命中 → 直接使用
  │   缓存未命中 → 东方财富搜索API (按公司名称)
  │
  ▼ 输出: 按时间线降序的新闻列表
```

### 新闻质量保障

- **按公司名称搜索**（非股票代码），提升新闻相关性
- **过滤泛资讯**：成交额/换手率/融资买入/融资余额/龙虎榜
- **按时间线降序**：方便回测按时间定位
- **WebSearch 缓存**：高质量新闻预填充，避免重复搜索
