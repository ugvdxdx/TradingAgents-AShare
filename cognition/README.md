# 项目认知 (Cognition)

这个文件夹记录选股系统的**实证认知**——通过回测验证、数据探索得出的结论,
以及对应的代码位置。每次有新的重要发现就追加一条日志。

## 为什么需要这个文件夹

选股系统的很多决策(锚公式、新晋股处理、boost 与否)都基于回测数据。
这些认知如果只存在对话历史里,换一个 agent 就丢了,容易导致:
- 重复验证已经否定过的方案(如"给新晋股加 boost")
- 遗忘已经成立的结论(如"delivery 在新晋股内是反向指标"——注: delivery 已于 2026-06-26 替换为 surge 爆发分, 此为历史示例)
- 代码和认知脱节(为什么 `_anchor_score` 是这个公式)

本文件夹是这些认知的**持久化沉淀**,供任何接手的 agent 快速对齐。

## 文件结构

- `findings.md` — 实证发现索引(按主题,每条附:结论 / 数据 / 代码位置 / 状态)
- `daily/` — 每日认知更新 (格式: `YYYY-MM-DD_简述.md`,记录当天新增的认知)

## 更新规则

1. **回测验证过的结论** → 写入 `findings.md` 对应主题,标注 Spearman/数据。
2. **每日新发现**(哪怕只是探索性结论)→ `daily/` 下新建日志。
3. **推翻旧结论** → 不要删除旧条目,标注"❌ 已推翻(by 日期/新数据)",保留推理过程。
4. 每条认知尽量附 `文件:行号`,方便直接定位代码。
